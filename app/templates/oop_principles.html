Calculator Program and Object Oriented Principles

The calculator program uses many of the object oriented principles

  1. Abstraction: Abstraction is the principle that creates a class with the intent to be inherited by other classes. This way certain methods and functions that will be run by default for each child class will already be once in the abstract class, so then the class can be implemented in different ways for each child class. For example, turning the tuple values into floats can be done in the abstract class so that it doesn't have to be written in each following class.
    2. Look at the example here.
  2. Inheritance: Inheritance is when a class inherits the methods of a parent class. This is done so that code isn't written repeatedly. For example, the calculation class will have a method that turns values into a float. So instead of re wrting this method several times in the addition and substraction classes, you can inherit the calculation method so that you can use that existing method and these new classes can now focus on writing unique methods.
    3. Look at the example here.
  3. Encapsulation: Encapsulation is the the priciple of restricting access to certain data. For example, in the Calculator program, the history class has a history list, which shouldn't be accessible to anyone outside of the class unless there is a method made with the ability to share desired pieces of that list.
    4. Look at the example here.
  4. Polymorphism: Polymorphsim is the ability to perform a function with multiple implementations. In the Calculator program, we have our calculation class that takes a list of values. We then have several child classes that inherits the constructor class and uses this list to return varying math results. Look at the calculation class and its child classes in full effect here.
