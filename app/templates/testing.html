AAA Testing

  • The AAA stands for Arrange, Act, and Assert
  • When testing for your code you need to
    1. Arrange inputs and targets.
    2. Act on target behavior, whether that is testing the outcome of a function, interacting with an API, etc.
    3. Assert expected outcomes, meaning to test for desired outcomes
  • This testing pattern is important because a program that passes a full coverage of tests will let future engineers know that the code works way after it is published.
  • Look at the following example from the calculator program. There is a test function for each function and class method in the Calc, History, and all the calculation classes. There are assertions looking for correct answers for each possible answer (if/else/different inputs).
