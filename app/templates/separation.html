Object Oriented Programming and the Calculator Program

Separation of Concerns

The calculator program utilizes the design pattern "Separation of Concerns" by separating the different utilites of a calculator:
  • By separating each type of calculation into separate classes, you can make changes to "addition" without affecting "subtraction".
  • Separating the history functionality into it's own class instead of putting it in the calculator class keeps related code and functions together.


  • Prior, the calculator class handled both calculations and history management, which was a lot code and functions that were not related.
  • Keeping conceptually similar functions together, it improves readability and makes it easier to keep track of what each class is doing.
  • Now the Calc class focuses on adding, subtracting, etc. and the History class focuses on extracting calculations in the list and getting results.
